// 
// 
// for문을 돌릴때 벡터와 리스트이 차이는

// cpu가 메모리로 접근하는 입출력은 비싸다
// 그래서 cpu는 본인안에 메모리를 저장하는데 그것을 캐시라고 한다
// 이 캐시가 벡터를 쓰는 진정한 이유가 나오는데 cache miss가 적기 때문이다
// 
// 리스트는 넥스트 포인터를 보고 가고 또 넥스트 포인터로 보고 접근을 하여 미미한 시간 접근이 있을 것이다

// 그런데 가용한 메모리가 있다고 하면 벡터는 그 가용한 메모리를 다쓰면 더 만들 수도 없지만
// 리스트는 다 쓰더라도 쓸 수 있는 메모리가 있다면 더 접근하여 쓸 수 있다는 것이다
// 
// 벡터는 24바이트라면 그런 배열들의 연속이겠지만
// 리스트는 24바이트 앞 뒤로 8바이트가 있어서 접근이 되는거다
// 그래서 이 중간에 있는 stl이 deque이다
