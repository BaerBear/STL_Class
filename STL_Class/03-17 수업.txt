C++과 최적화 기법을 심도 있게 다루는 이번 STL(Standard Template Library) 수업은 게임 프로그래밍과 엔진 최적화에도 곧바로 적용할 수 있는 아주 핵심적인 내용이네요! 교수님께서 강조하신 중요한 포인트들을 이해하기 쉽게 구조화하여 정리해 드립니다.
1. 키워드(Keyword)와 식별자(Identifier)
2. 코딩을 할 때 흔히 쓰는 main은 C++의 예약된 키워드가 아닙니다. 반면 and, or 같은 단어들은 키워드에 해당합니다.
3. 우리가 마음대로 이름을 붙일 수 있는 변수명 등은 '식별자(Identifier)'라고 부릅니다.
4. 컴파일러는 내부적으로 이러한 변수명들을 모두 메모리 주소(숫자)로 변환하여 처리합니다.
5. std::array와 템플릿(Template)의 동작 원리
6. 기존 C 스타일의 배열 대신 C++ 표준인 std::array를 사용하는 것이 강력히 권장됩니다. 기존 배열에 비해 속도가 느려지거나 메모리를 더 쓰는 등의 손해가 전혀 없기 때문입니다.
7. 템플릿의 인자로는 데이터 타입(Type)뿐만 아니라 숫자(Value), 다른 템플릿, 그리고 갯수가 정해지지 않은 가변 인자(Variadic)도 전달될 수 있습니다.
8. 컴파일러는 템플릿 코드를 만나면 넘겨받은 타입과 숫자에 맞춰 실제 동작할 수 있는 코드를 내부적으로 찍어내는 '특수화(Specialization)' 작업을 수행합니다.
9. 파일 입출력(File I/O)과 개행문자 최적화
10. 1,000만 개의 난수(int)를 파일에 저장하는 실습을 통해 출력 속도의 차이를 비교했습니다.
11. 데이터를 출력할 때 std::endl을 사용하면 내부적으로 버퍼를 비우는 플러싱(Flushing) 작업이 동반되어 속도가 심각하게 느려집니다.
12. 따라서 대량의 데이터를 다루거나 코딩 테스트를 볼 때는 \n (백슬래시 n)을 사용하는 것이 훨씬 유리합니다.
 13. C++은 RAII(Resource Acquisition Is Initialization) 원칙을 따르기 때문에, 스트림 객체(ifstream 등)의 수명이 다하면 해당 파일 자원과의 연결이 자동으로 안전하게 해제됩니다.
14. STL 알고리즘과 이터레이터(Iterator)를 활용한 최적화 (수업의 핵심)
15. 과제: 파일에 저장된 1,000만 개의 정수 중 최솟값을 찾아야 합니다.
16. 잘못된 접근 (0점짜리 답안): 파일 데이터를 모두 읽어서 40MB가 넘는 거대한 메모리(배열이나 벡터)에 전부 담은 뒤에 알고리즘을 돌리는 것은 불필요한 메모리를 심각하게 낭비하는 방식입니다.
17. 올바른 STL 접근법: std::min_element 같은 알고리즘은 자료형에 종속되지 않게 추상화되어 있습니다. 따라서 데이터를 메모리에 다 올릴 필요 없이, 입력 스트림 자체를 순회할 수 있는 스트림 이터레이터(istream_iterator)를 사용하면 됩니다.
18. 알고리즘에 스트림 이터레이터의 시작과 끝을 넘겨주면, 파일을 단 한 번만 읽어 내려가면서 즉시 최솟값을 찾아낼 수 있어 메모리를 획기적으로 절약할 수 있습니다.
마지막에 다룬 스트림 이터레이터와 std::min_element를 결합한 코드는 실제 C++ 실무에서도 굉장히 우아하게 쓰이는 기법입니다. 혹시 이 논리를 바탕으로 한 줄로 완성되는 1,000만 개 최솟값 찾기 C++ 코드를 직접 작성해서 보여드릴까요?
